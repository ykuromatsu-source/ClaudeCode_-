# 大濠うなぎ SNS自動運用エージェント

**「うなぎ文化をもっと身近に。」** — 2025年8月オープンの大濠うなぎ専門店に特化した、Advisor-Executor協調型のSNSコンテンツ自動生成エージェントです。

単なる「メニューPR文の自動生成」に留まらず、団体利用・テイクアウト・ランチ・ディナー・コース紹介・地域密着情報・鰻の豆知識という7つの切り口でブランドの世界観を多角的に発信し、曜日別の自動スケジューリング、カレンダー管理、一括エクスポートまでを一気通貫でこなす、実運用を見据えた設計になっています。

## 概要

大濠うなぎは、大濠のお堀周辺が持つ歴史的背景を現代に再解釈した専門店です。高級店のような堅苦しさはなく、質は極めて高いが親しみやすい——そんなブランドの空気感を、SNS投稿の一文一文まで一貫して再現することを目指しています。

このエージェントは、既存の高品質スキル資産（`insta-food-buzz` — トリガー評価100%達成済み）が確立した「五感描写」「冒頭フック」「自然なCTA」といった品質基準を核として読み込み、それを土台に **Worker（Sonnet 5想定）が起草し、Advisor（Fable 5想定）がレビューする** Advisor-Executorパイプラインでドラフトを生成します。

## コア機能一覧

| 機能 | 説明 |
|---|---|
| 🍱 **7つのコンテンツカテゴリ生成** | メニュー訴求・団体向け（宴会）・テイクアウト・ランチ用・ディナー用・コース紹介・地域密着＆周辺紹介・鰻の豆知識の多角的なバリエーションを生成 |
| 🏷️ **ハッシュタグ自動最適化** | 「共通タグ→カテゴリ別プリセット→AI生成タグ」の優先順位で自動的に組み合わせ、重複除去・上限適用まで一括処理 |
| 📅 **カレンダー自動ストック** | 生成したドラフトを `outputs/YYYY-MM-DD/` 配下へ投稿予定日ごとに自動整理・保存 |
| 📖 **ストックログ閲覧（`list`）** | 保存済みドラフトを月・日付で絞り込み、月ごとにグループ化した見やすい一覧をコンソール表示 |
| 🗓️ **曜日別自動シミュレーター（`simulate`）** | 曜日ごとに最適なカテゴリを自動割り当てし、指定期間分をワンコマンドで一括生成・自動保存 |
| 📦 **一括エクスポート（`export`）** | バラバラに保存されたドラフト群を、投稿予定日順に結合した1つのMarkdownファイルへ集約 |

## アーキテクチャ

```
                 ┌─────────────────────────┐
                 │  skill_knowledge.py      │  insta-food-buzz の
                 │  (読み取り専用)           │  品質基準を読み込み
                 └────────────┬─────────────┘
                              │
┌──────────────┐   ┌──────────▼───────────┐   ┌──────────────────┐
│  main.py     │──▶│  content_service.py   │◀──│  run_demo.py      │
│ (live優先)    │   │  BrandRules           │   │ (mockフォールバック)│
│              │   │  ContentCategory      │   │                  │
└──────────────┘   │  Advisor-Executor統合 │   └──────────────────┘
                    │  ハッシュタグ最適化    │
                    │  カレンダー保存/一覧/  │
                    │  シミュレーター/       │
                    │  エクスポート          │
                    └──────────┬────────────┘
                               │
                    ┌──────────▼────────────┐
                    │  agent_core.py         │
                    │  AdvisorExecutorAgent  │
                    │  (Worker ⇄ Advisor)    │
                    └────────────────────────┘
```

- **`main.py`**: 実API呼び出し（`ANTHROPIC_API_KEY` 必須）を優先するエントリポイント。
- **`run_demo.py`**: APIキー未設定時、または実行に失敗した場合に自動でモックモードへフォールバックするデモ実行スクリプト。**このREADMEの実行例はすべて `run_demo.py` を前提**にしています（モック環境でも100%動作するため）。
- **`content_service.py`**: ブランドルール・コンテンツカテゴリ定義・Advisor-Executorパイプラインの組み立て・ハッシュタグ最適化・カレンダー保存/一覧/シミュレーター/エクスポートといった、システムの中核ロジック一式。

## セットアップ方法

### 1. リポジトリのクローン

```bash
git clone <このリポジトリのURL>
cd sns-auto-agent
```

### 2. 依存パッケージのインストール

```bash
pip install -r requirements.txt
```

### 3. `.env` の設定（モック／実APIの切り替え）

`sns-auto-agent/.env` にAPIキーを1行追加すると実API呼び出しモードになります（`.env` は `.gitignore` で除外済みなので、リポジトリにはコミットされません）。

```bash
# sns-auto-agent/.env
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxx
```

| 状態 | 挙動 |
|---|---|
| `.env` にキーが**設定されている** | `run_demo.py` は実APIを呼び出し（`live`モード）。呼び出しに失敗した場合は自動的にモックへフォールバック |
| `.env` にキーが**未設定** | 全コマンドが自動的にモックモードで動作し、パイプライン構造・出力フォーマットのみを決定的なプレースホルダー値で再現する（AI生成であるかのように偽装しない） |

`.env` を使わず、同一プロセス内で環境変数として渡すことも可能です。

```bash
export ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxx
python3 run_demo.py
```

## 使い方 (Usage)

### 単体カテゴリの生成（例: ランチ用を指定日で生成）

```bash
python3 run_demo.py lunch --date 2026-07-12
```

`instagram_caption` / `instagram_hashtags`（自動最適化済み）/ `line_message` / `x_post` / `threads_post` / `image_prompt_en` の6種ドラフトがMarkdownで出力され、同時に `outputs/2026-07-12/unagi_lunch.md` へ自動保存されます。

### 全7カテゴリを一括生成

```bash
python3 run_demo.py all --date 2026-07-15
# 引数を省略すると category=all, date=実行当日 として扱われる
python3 run_demo.py
```

### ストックログの確認（`list`）

```bash
# 保存済みの全ストックを一覧表示
python3 run_demo.py list

# 年月で絞り込み
python3 run_demo.py list --month 2026-07

# 投稿予定日で絞り込み
python3 run_demo.py list --date 2026-07-12
```

出力例:

```
# 2026-07
- 📅 2026-07-12｜【大濠うなぎ】ランチ用
    生成日時: 2026-07-11 13:36:04
    ファイル: 2026-07-12/unagi_lunch.md

合計 1 件
```

### 曜日別自動シミュレーター（`simulate`）

曜日ごとに最適なカテゴリを自動で割り当て、指定期間分を一括生成・自動保存します。

```bash
# 実行当日から1週間分を自動生成
python3 run_demo.py simulate --days 7

# 開始日・期間を指定
python3 run_demo.py simulate --start-date 2026-08-01 --days 3
```

| 曜日 | 自動割り当てカテゴリ | 狙い |
|---|---|---|
| 月 | ランチ用 | 週始めのご褒美ランチ需要 |
| 火 | コース紹介 | 定番・コースラインナップの網羅紹介 |
| 水 | テイクアウト | 週半ばの持ち帰り需要 |
| 木 | 鰻の豆知識 | エンタメ性のある読物コンテンツ |
| 金 | 団体向け（宴会） | 週末に向けた宴会・飲み放題訴求 |
| 土 | ディナー用 | 週末のゆったりとしたディナー |
| 日 | 地域密着・周辺紹介 | 休日のお出かけ・大濠公園周辺情報 |

### 一括エクスポート（`export`）

`outputs/` にバラバラに保存されたドラフトを、投稿予定日順に結合した1つのMarkdownファイルへまとめます。週刊・月刊のまとめ資料作成に。

```bash
# 全期間を outputs/combined_export.md へ結合
python3 run_demo.py export

# 年月で絞り込み、出力先を指定
python3 run_demo.py export --month 2026-07 --out outputs/2026-07-まとめ.md
```

### コマンド早見表

| コマンド | 説明 |
|---|---|
| `run_demo.py [category] [--date YYYY-MM-DD]` | 指定カテゴリ（省略時 `all`）を指定日（省略時は当日）で生成・保存 |
| `run_demo.py list [--month YYYY-MM] [--date YYYY-MM-DD]` | ストック済みドラフトの一覧表示 |
| `run_demo.py simulate [--start-date YYYY-MM-DD] [--days N]` | 曜日別ルールで期間一括生成（既定7日間） |
| `run_demo.py export [--month YYYY-MM] [--out PATH]` | ストックを1ファイルへ結合エクスポート |

`main.py` も同じサブコマンド体系（`list` / `simulate` / `export`）に対応しており、実APIキーが設定されていればそのまま実運用のエントリポイントとして使えます。

## プロジェクト構成

```
sns-auto-agent/
├── .env                    # APIキー設定（gitignore対象、リポジトリには含まれない）
├── requirements.txt
├── agent_core.py           # Advisor-Executorパイプラインの核（Worker/Advisor協調ロジック）
├── content_service.py      # ブランドルール・カテゴリ定義・生成/保存/一覧/シミュレーター/エクスポート
├── skill_knowledge.py      # insta-food-buzzスキル資産の読み取り専用ローダー
├── main.py                 # 実APIエントリポイント
├── run_demo.py             # モックフォールバック付きデモ実行スクリプト
├── test_agent_core.py      # 単体テスト（モックベース、APIキー不要）
└── outputs/                # 生成済みドラフトの自動保存先（YYYY-MM-DD/事業_カテゴリ.md）
```

## 品質へのこだわり

- **insta-food-buzzの知見を継承**: トリガー評価で正解率100%を達成済みの既存Instagram投稿生成スキルが確立した五感描写・スコアリング基準・CTAフォーマットを、そのまま生成の核として踏襲しています。
- **モック環境での完全動作保証**: APIキー未設定・実API呼び出し失敗のいずれの状況でも、パイプラインの制御フロー（計画→レビュー→生成→品質レビュー→自律修正）と出力フォーマットを決定的なプレースホルダー値で再現し、クラッシュしない堅牢性を最優先しています。
- **CLIバリデーション**: `--date`（YYYY-MM-DD）・`--month`（YYYY-MM）・`--days`（正の整数）はいずれも不正入力時に使い方メッセージを表示して安全に終了します。

```bash
# テストの実行
python3 -m unittest test_agent_core.py -v
```
